## What Makes This Repo Different

<div class="grid cards" markdown>

-   :material-layers-triple-outline: __Curriculum first__

    The goal is a coherent ladder from foundations to advanced topics, not a random pile of solved problems.

-   :material-brain: __Proof-aware teaching__

    Topic pages are meant to explain `when to use it`, `why it works`, and `how to code it cleanly`.

-   :material-flag-checkered: __Contest-shaped practice__

    Practice lives near its topic, so readers can move from concept to implementation to problems without context switching.

-   :material-link-variant: __Source-backed reading__

    Each serious topic should link outward to courses, editorials, references, and deeper theory for motivated readers.

</div>

## Start In 15 Minutes

<div class="grid cards" markdown>

-   __1. Pick your current level__

    Use the staged ladder in the project overview to decide whether you should start with foundations, core data structures, or contest-ready topics.

    [See the progression ladder](project-readme.md#progression-ladder)

-   __2. Commit to one topic family__

    Choose a lane like graphs, DP, strings, or math and stay there long enough to build recognition, not just exposure.

    [Open topic sections](topics/README.md)

-   __3. Solve one anchored problem__

    After reading the idea, solve a targeted problem note and compare it with a clean repository solution.

    [Go to practice ladders](practice/README.md)

</div>

!!! tip "Best way to use this site"
    Read one topic page, solve one problem from the matching ladder, then write or adapt one clean C++ implementation.
    That loop is what turns reading into contest skill.

## Long-Term Vision

This site should eventually feel like a compact, open curriculum for competitive programming:

- concise enough to skim before practice
- rigorous enough to support serious learning
- visual enough to feel inviting instead of academic and dry
- linked enough that deeper exploration is always one click away
