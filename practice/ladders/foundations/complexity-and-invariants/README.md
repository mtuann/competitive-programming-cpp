# Complexity And Invariants Ladder

This ladder is not about one algorithm. It is about learning to justify and debug your implementation.

## Warm-Up

- estimate the complexity of a simple nested loop
- explain what each pointer or DP state means

## Core

- binary search invariant checks
- prefix-sum correctness arguments
- prove why a greedy-after-sorting step is safe

## Stretch

- write a short proof sketch before coding
- restate one accepted solution as invariants plus transitions only
