# IOI / OI Playbook

## What Matters Most

- deeper modeling
- proof confidence
- stronger DP, graph, and combinatorics coverage

## Training Rhythm

- fewer problems, more post-solve review
- rewrite hard solutions cleanly after learning them
- keep a list of invariants and proof patterns that you missed

## Before A Contest

- revisit harder topic pages rather than random warm-ups
- re-check notebook snippets you can actually code from memory

## After A Contest

- fully understand one editorial-quality path for each missed task
- record whether the blocker was modeling, proof, or implementation
